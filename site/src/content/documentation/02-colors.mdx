# Colors

Our color system uses a three-layer architecture: foundation colors, semantic tokens, and theme-specific values.

## Foundation Colors

Foundation colors are base color values defined as HSL hue and saturation pairs:

- **Burgundy** (`--f-color-burgundy`): Accent color at 356.7° hue, 94.8% saturation
- **Gray** (`--f-color-gray`): Neutral grays at 0° hue, 0% saturation
- **Orange** (`--f-color-orange`): Warm tones at 38° hue, 100% saturation
- **Red** (`--f-color-red`): Danger states
- **Green** (`--f-color-green`): Success states
- **Blue** (`--f-color-blue`): Info states

## Semantic Tokens

Semantic tokens reference foundation colors and adjust lightness for specific UI purposes:

- **Text**: `--color-text`, `--color-text-weak`, `--color-text-weaker`, `--color-text-weakest`
- **Surfaces**: `--color-background`, `--color-surface`, `--color-surface-raised`, `--color-surface-lowered`
- **Borders**: `--color-border`, `--color-border-strong`, `--color-border-hover`
- **Status**: `--color-status-danger`, `--color-status-success`, `--color-status-warning`, `--color-status-info`, `--color-status-neutral`
- **Accent**: `--color-accent` (burgundy at 35% lightness)

## Theme Support

All semantic tokens automatically adapt to light and dark themes by adjusting their lightness values while maintaining the same foundation colors.
