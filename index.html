Colors

Describes the color system used in the design.

Accent Colors

Documentation... todo

Color Name Description
color-accent Accent color. Used as primary button background, hero area background, and as a link color. This color provides accessible contrast when used for text on light background.

Text Colors

Documentation... todo

Color Name Description
color-text Default text color. Used as the main text color throughout the UI. Provides good and legible contrast for our content.
color-text-link Default link text color. Used for all paragraph level links in an user interface.
color-text-weak Weak text color. Used for e.g. introduction paragraphs and help texts that are coupled with form elements.
color-text-weaker Weaker text color. Used for e.g. disabled buttons, footers, table headings, captions and placeholder texts.
color-text-weakest Weakest text color. Used for e.g. disabled input text and placeholder text in focused inputs.
color-text-on-accent Default text on accent color. Used when text sits on the accent color or a dark background.
color-text-danger Default text color for danger messages. Also used for the danger badge text.
color-text-success Default text color for success messages. Also used for the success badge text.
color-text-neutral Neutral text color. Used for the neutral badge text.
color-text-info Info text color. Used for the info badge text.
color-text-warning Warning text color. Used for the warning badge text.

Background & Surface Colors

Documentation... todo

Color Name Description
color-background Default application background color. Should be set to either html or body element.
color-surface Default application surface color. Use this for e.g. card backgrounds.
color-surface-raised Default raised surface color. Use this when you need to separate a specific layer from the surface color, e.g. table headers.
color-surface-lowered Default lowered surface color. Use this when you need to make a specific layer to appear lower than the surface color, e.g. progress bar.

Border Colors

Documentation... todo

Color Name Description
color-border Default border color
color-border-strong Strong border color for highlighting specific parts in the user interface.
color-border-hover Default border hover color.
color-border-danger Danger border color. Used for the danger badge border style.
color-border-success Success border color. Used for the success badge border style.
color-border-neutral Neutral border color. Used for the neutral badge border style.
color-border-info Info border color. Used for the info badge border style.
color-border-warning Warning border color. Used for the warning badge border style.

Status Colors

Documentation... todo

Color Name Description
color-status-danger Default danger status color. Used to communicate problems that have to be resolved so that user can continue forward. This color should be always used for e.g. highlighting errors.
color-status-success Default success status color. Used to convey success states. For example, you might show a success message that tells the user that changes were saved or something was completed.
color-status-neutral Default neutral status color.
color-status-info Default info status color. Used to convey something informal.
color-status-warning Default warning status color. Used to display information that needs user’s attention and may require further steps.
color-status-danger-weak Weak danger status color. Used to communicate problems that have to be resolved so that user can continue forward. This color should be always used for e.g. highlighting errors.
color-status-success-weak Weak success status color. Used to convey success states. For example, you might show a success message that tells the user that changes were saved or something was completed.
color-status-neutral-weak Weak neutral status color. Used for the badge default state.
color-status-info-weak Weak info status color. Used to convey something informal.
color-status-warning-weak Weak warning status color. Used to display information that needs a user’s attention and may require further steps.
