The color system transforms seed colors into purpose-driven design tokens that automatically adapt to light and dark themes. Instead of using raw color values, you use semantic tokens that describe the intent of the color.

---

## How it works

### 1. Seed Colors

The system starts with four seed colors that define the base hue for each color family:

- `--color-primary-seed` - Main brand color (green)
- `--color-neutral-seed` - Grayscale colors (#8b8c93)
- `--color-constructive-seed` - Success/positive actions (#5dbb55)
- `--color-destructive-seed` - Error/negative actions (#ef5655)

### 2. Color Scale Generation

Each seed color is automatically expanded into an 11-step scale (50-950) using `color-mix()` in oklab color space:

- **50-400**: Progressively lighter tints (mixed with white)
- **500**: The seed color itself
- **600-950**: Progressively darker shades (mixed with black)

Example:
```css
--color-primary-500: var(--color-primary-seed);
--color-primary-300: color-mix(in oklab, var(--color-primary-seed), white 47%);
--color-primary-700: color-mix(in oklab, var(--color-primary-seed), black 28.25%);
```

### 3. Semantic Token Mapping

The scale values are then mapped to semantic tokens that describe their purpose. These mappings flip between light and dark themes to maintain proper contrast.

## Token Categories

### Fill Colors

Background colors for components and surfaces. Each color family (primary, neutral, constructive, destructive) has five fill intensities:

- `--color-{family}-fill-softer` - Very subtle backgrounds
- `--color-{family}-fill-soft` - Subtle backgrounds  
- `--color-{family}-fill-mid` - Medium emphasis backgrounds
- `--color-{family}-fill-loud` - High emphasis backgrounds
- `--color-{family}-fill-louder` - Maximum emphasis backgrounds

Light theme: Uses lighter scale values (100-900)  
Dark theme: Uses darker scale values (900-100, inverted)

### Text Colors

Text that appears on different background intensities. Automatically paired to ensure readable contrast:

- `--color-{family}-text-on-soft` - Text on soft backgrounds
- `--color-{family}-text-on-mid` - Text on medium backgrounds
- `--color-{family}-text-on-loud` - Text on loud backgrounds
- `--color-{family}-text-colorful` - Colorful accent text on neutral backgrounds

### Stroke Colors

Border and outline colors for each family:

- `--color-{family}-stroke-softer` - Very subtle borders
- `--color-{family}-stroke-soft` - Subtle borders
- `--color-{family}-stroke-mid` - Medium emphasis borders

## General Purpose Tokens

### Text

Generic text colors that work on the page background:

- `--color-text` - Primary body text (70% black in light, 60% white in dark)
- `--color-text-mid` - Secondary text (50% mix)
- `--color-text-softer` - Tertiary text (30% mix)  
- `--color-text-link` - Hyperlink text

### Backgrounds

Page and surface colors:

- `--color-background` - Main page background
- `--color-surface` - Raised surface (cards, panels)
- `--color-surface-raised` - Elevated surface (modals, popovers)

### Strokes

Generic border colors:

- `--color-stroke` - Default borders
- `--color-stroke-strong` - Emphasized borders
- `--color-stroke-hover` - Interactive element borders

## Usage Examples

### Colored Button

```css
.button-primary {
  background: var(--color-primary-fill-mid);
  color: var(--color-primary-text-on-mid);
  border: 1px solid var(--color-primary-stroke-mid);
}

.button-primary:hover {
  background: var(--color-primary-fill-loud);
  color: var(--color-primary-text-on-loud);
}
```

### Subtle Badge

```css
.badge-success {
  background: var(--color-constructive-fill-softer);
  color: var(--color-constructive-text-on-soft);
  border: 1px solid var(--color-constructive-stroke-softer);
}
```

### Error Message

```css
.error-message {
  background: var(--color-destructive-fill-soft);
  color: var(--color-destructive-text-on-soft);
  border-left: 3px solid var(--color-destructive-stroke-mid);
}
```

## Theme Adaptation

All semantic tokens automatically adapt when switching between `[data-theme="light"]` and `[data-theme="dark"]`. You never need to write theme-specific code - just use the semantic tokens and the system handles the rest.

The inversion ensures:
- Proper contrast ratios are maintained
- Visual hierarchy remains consistent
- Colors feel appropriate for each theme

## Benefits

1. **Consistent contrast**: Text/fill pairings are pre-calculated for accessibility
2. **Automatic theming**: Change themes without touching component code
3. **Semantic naming**: Token names describe purpose, not appearance
4. **Flexible customization**: Change seed colors to rebrand the entire system
5. **Reduced decisions**: Fewer choices lead to more consistent designs
