Design tokens are organized in a two-tier system that separates raw values from their intended usage.

## Foundation Tokens
Foundation tokens are the lowest-level building blocks that specify concrete values. They should be named descriptively based on what they are, not how they’re used.

**Examples:** `--shadow-1`, `--size-4`, `--radius-2`

## Semantic Tokens
Semantic tokens describe how and where a value should be applied. They map foundation tokens to specific use cases and UI patterns.

**Examples:** `--color-text`, `--color-accent-fill`, `--radius-input`

> **Key Principle:** Don’t create semantic tokens “just because.” Create them when they encode a decision or pattern you want to maintain. The semantic layer should be lighter than you think—it’s not a 1:1 mapping of foundation tokens with different names.

### Theming
The semantic layer is where theming happens. Any value that should be themable must be defined as a semantic token. This allows you to swap themes by changing semantic token values while keeping your foundation tokens stable.

For example, light and dark themes would redefine semantic 
tokens like `--color-background` and `--color-text` to 
different foundation values, but the foundation tokens 
themselves (`--color-gray-seed`, `--color-green-seed`, etc.) remain constant.

## Using Tokens
When styling UI elements, use both semantic and foundational tokens. 
Apply semantic tokens to theme-customizable properties like colors, 
borders, and shadows, while using foundational tokens for fixed 
values that remain consistent across themes.

### Example
```css
/* Foundation */
--size-4: 1rem;
--size-8: 2rem;
--radius-2: 0.25rem;
--radius-3: 0.5rem;

/* Semantic (only where patterns exist) */
--spacing-stack-small: var(--size-4);
--spacing-stack-large: var(--size-8);
--radius-interactive: var(--radius-2);

/* Component */
.button {
  padding: var(--size-2) var(--size-4); /* Direct foundation use */
  border-radius: var(--radius-interactive); /* Semantic pattern */
  gap: var(--spacing-stack-small); /* Semantic pattern */
}
```
