## Buttons

The button system provides a systematic approach to interactive elements
with clear visual hierarchy through variants, styles, sizes, and states.
Designed for terminal-like immediacy with instant state changes—no
transitions.

### Variants & Styles

**Variants:**

- **Primary** (default): Filled background, strong presence
- **Secondary**: Outline style (`data-variant="secondary"`)

**Styles:**

- **Primary** (default): Terminal green - the default button color
- **Neutral**: Neutral gray (`data-style="neutral"`)
- **Constructive**: Success green (`data-style="constructive"`)
- **Destructive**: Error red (`data-style="destructive"`)

### Sizes

- **Compact**: Reduced padding (`data-size="compact"`)
- **Default**: Standard size (no attribute)
- **Large**: Generous padding (`data-size="large"`)

<CodeExample
  html={`<div style="display: flex; gap: var(--size-2); flex-wrap: wrap; align-items: center;">
  <button class="button" data-style="primary" data-size="compact">Compact</button>
  <button class="button" data-style="primary">Default</button>
  <button class="button" data-style="primary" data-size="large">Large</button>
</div>`}
/>

All buttons support hover, active, focus, and disabled states with instant
transitions.

  <button class="button" data-style="primary" aria-disabled="true">Aria Disabled</button>

/>

### Accessibility

- Minimum 44×44px touch target (met even with compact size)
- Keyboard navigation with visible focus outline
- Disabled state uses both `disabled` attribute and `aria-disabled` support
- Icon-only buttons require `aria-label` for screen readers

## Typesetting

The `.typesetting` class creates consistent vertical rhythm for text-heavy
content using CSS Grid. It styles headings, paragraphs, code blocks,
horizontal rules, and maintains proper spacing between all elements.

### Features

- All heading levels (h1-h6) with appropriate sizing
- Inline code and code blocks with styled borders
- Horizontal rules for section breaks
- Consistent vertical rhythm using `1rlh` gap
- Extra spacing before headings for visual hierarchy

### Use Cases

Apply `.typesetting` to containers with:

- Articles and blog posts
- Documentation pages
- Long-form written content
- Any content requiring consistent vertical rhythm
