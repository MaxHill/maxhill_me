# Components

Reusable components ensure consistency and accelerate development.

## Buttons

Buttons trigger actions and come in multiple variants for different contexts.

## Forms

Form elements follow consistent styling patterns and provide clear feedback states.

## Navigation

Navigation components help users move through the interface efficiently.

## Cards

Cards group related content and actions in a contained, scannable format.

<card-element>
  <div slot="title">Card Element Title</div>
  <p>This is the main content of the card element.</p>
  <p>It uses web components with shadow DOM.</p>
  <div slot="footer">Card element footer</div>
</card-element>

<card-element data-variant="outline">
  <div slot="title">Outline Variant</div>
  <p>This card element has a outline variant.</p>
  <div slot="footer">Footer info</div>
</card-element>

<card-element>
  <div slot="title">Card Element with Title Only</div>
  <p>This card element has a title but no footer.</p>
</card-element>

<card-element>
  <p>Card Element with Content Only</p>
  <p>No title or footer, just content.</p>
</card-element>

## Tabs

Tab containers provide accessible navigation between related content panels.

<tab-container>
  <div role="tablist">
    <button type="button" role="tab" aria-selected="true">Tab 1</button>
    <button type="button" role="tab">Tab 2</button>
    <button type="button" role="tab">Tab 3</button>
  </div>
  <div role="tabpanel">
    <p>Content for tab 1</p>
  </div>
  <div role="tabpanel" hidden>
    <p>Content for tab 2</p>
  </div>
  <div role="tabpanel" hidden>
    <p>Content for tab 3</p>
  </div>
</tab-container>

For full documentation, see the [official tab-container-element docs](https://github.github.com/tab-container-element/examples/).
