## Examples

### Basic Usage

A search list that filters items using fuzzy search.

<m-search-list>
  <input slot="controller" type="search" placeholder="Search programming languages..." />
  <div data-keywords="javascript js ecmascript">JavaScript</div>
  <div data-keywords="typescript ts">TypeScript</div>
  <div data-keywords="python py">Python</div>
  <div data-keywords="rust rs">Rust</div>
  <div data-keywords="go golang">Go</div>
  <p slot="empty">No matching languages found</p>
  <p slot="initial">Start typing to search programming languages</p>
</m-search-list>

## Usage Notes

### Search Algorithm

The component uses fuzzy search to match items. It searches through:
- The item's text content
- The `data-keywords` attribute value

### Accessibility

- Includes ARIA live region that announces search results to screen readers
- Announces result count after 1 second of inactivity
- Properly manages focus and keyboard interaction through the input element

### CSS States

The component exposes three custom states:
- **`:state(initial)`**: No search query has been entered
- **`:state(match)`**: A search query is entered and has matches
- **`:state(empty)`**: A search query is entered but has no matches

### Slots

- **`controller`**: Input element that controls the search
- **default**: Items to be filtered
- **`empty`**: Content shown when search returns no results
- **`initial`**: Content shown before any search is performed

### Data Attributes on Items

After filtering, items receive a `data-match` attribute:
- `data-match="true"`: Item matches the search
- `data-match="false"`: Item does not match the search
